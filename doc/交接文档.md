## 1. 项目是什么

面向非法律专业用户的合同智能解读 App（鸿蒙高校创新赛 · 应用创新）。

- **价值**：合同摘要 → 红黄绿风险分级 → 原文对照白话解释与修改建议
- **形态**：HarmonyOS 原生客户端（ArkTS/ArkUI）+ 同仓 Node.js 分析服务
- **首版不做**：电子签署、律师咨询、合同改写、多人协作、法律意见书导出

## 4. 架构与协议

```text
客户端 ──上传/轮询/取结果──▶ server(:3000)
                              ├─ OCR / 文本清洗 / 条款切分
                              ├─ LLM 结构化分析
                              └─ 风险分级与摘要
客户端本地：Preferences 缓存、接续、卡片、隐私窗
```

### API（首版固定）

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/v1/contracts/analyze` | 创建任务，返回 `taskId` |
| GET | `/v1/tasks/{taskId}/status` | `PENDING \| OCR_RUNNING \| LLM_RUNNING \| SUCCESS \| FAILED` |
| GET | `/v1/tasks/{taskId}/result` | 返回 `AnalysisResult` |

### 核心数据契约

```ts
type RiskLevel = 'RED' | 'YELLOW' | 'GREEN'

interface ClauseRisk {
  clauseId: string
  title: string
  originalText: string
  plainText: string
  riskLevel: RiskLevel
  riskReason: string
  suggestion?: string
  anchors: { page?: number; paragraph?: number }
}

interface AnalysisResult {
  taskId: string
  contractName: string
  overallSummary: string
  signBeforeChecklist: string[]
  riskStats: { red: number; yellow: number; green: number }
  clauses: ClauseRisk[]
  generatedAt: string
}
```

客户端不直连大模型；模型调用统一在服务端。

## 5. 如何跑起来

### 5.1 后端

```bash
cd server
cp .env.example .env   # 填入 LLM_API_KEY（或使用 apikey.txt）
npm install
npm run dev            # 默认 http://0.0.0.0:3000
```

健康检查：`curl http://127.0.0.1:3000/health`

默认模型配置见 `server/.env.example`（阿里云百炼 OpenAI 兼容端点，`qwen-plus`）。

### 5.2 客户端

1. DevEco Studio 打开仓库根目录，运行 `entry` 模块到模拟器/真机。
2. 默认服务地址：`http://127.0.0.1:3000`（见 `common/constants.ets` 的 `API_BASE_URL`）。
3. **真机联调**：把服务地址改成电脑局域网 IP，不要用 `127.0.0.1`；也可在导入页填写。
4. API Key 也可在 App **设置页**本地保存并透传。

### 5.3 无后端时的演示兜底

导入页选「高风险样例 / 低风险样例」或「直接演示当前样例」；设置页「答辩演示模式」可选弱网 / 超时 / 服务失败 / 降级展示。

联调也可喂文本：`inline://合同正文…`。

## 6. 答辩演示（约 3 分钟）

推荐顺序：

1. **开场**：痛点（看不懂合同）→ 三层信息（摘要 / 风险 / 白话对照）
2. **高风险样例**：导入 → 分析中 → 结果总览 → 点进条款对照（原文 / 白话 / 原因 / 建议）
3. **（可选）低风险样例**：证明系统有区分能力
4. **鸿蒙三项**：设置页隐私防窥；口头/实机讲应用接续；桌面卡片或首页最近记录
5. **（可选）异常模式**：超时 / 服务失败 / 降级展示
6. **收尾**：AI 主链路 + 鸿蒙全场景能力

现场求稳：优先「高风险样例 + 降级展示」。

## 7. 现场预案（一页版）

| 问题 | 立刻怎么做 |
|------|------------|
| 后端挂了 / 无网 | 设置页切「降级展示」或「服务失败」，用本地样例 |
| 分析太慢 | 切「弱网」讲反馈；或直接用样例跳过 |
| 模型超时 | 切「超时」，展示失败态与重试 |
| 接续演示不了 | 讲 payload 已落地；用最近记录 / 结果页说明同一恢复能力 |
| 卡片不好使 | 用首页最近记录展示同一快捷回看 |
| 隐私效果不明显 | 展示设置开关 + 敏感页调用链路 |

答辩前 5 分钟自检：服务地址是否局域网 IP、高风险样例能否直出、降级模式能否进结果页、隐私开关可切。
