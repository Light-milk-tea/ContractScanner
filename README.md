# 法言白话

面向非法律专业用户的合同智能解读 App。  
鸿蒙高校创新赛 · 应用创新赛道。

签合同前先看懂重点：摘要 → 红黄绿风险 → 原文对照白话解释与修改建议。

---

## 它解决什么问题

租房、实习、培训、服务协议里，普通用户常遇到：

- 条款又长又专业，读不完、看不懂
- 风险藏在字缝里，签完才发现
- 不知道哪些条款必须重点确认

《法言白话》不替代律师，而是帮你先「看明白、看重点、看风险」。

---

## 核心能力

| 能力 | 说明 |
|------|------|
| 合同摘要 | 先给整体结论与签署前确认项 |
| 风险分级 | 红 / 黄 / 绿，快速定位重点 |
| 对照阅读 | 原文 · 白话 · 风险原因 · 修改建议 |
| 分层阅读 | 总览 → 风险列表 → 单条精读 |

### 主链路

```text
导入合同 → 分析中 → 结果总览 → 条款对照阅读
```

### 鸿蒙特性

- **应用接续**：手机分析，平板 / PC 续看结果
- **互动卡片**：桌面快捷回看最近合同摘要
- **隐私防窥**：敏感页限制截屏 / 录屏

## 技术架构

```text
HarmonyOS 客户端 (ArkTS / ArkUI)
        │  上传 / 轮询状态 / 取结果
        ▼
Node.js 分析服务 (:3000)
        ├─ OCR / 文本清洗 / 条款切分
        ├─ 大模型结构化分析（OpenAI 兼容接口）
        └─ 风险分级与摘要输出
```

客户端不直连大模型；模型调用统一在服务端。

| 层级 | 技术 |
|------|------|
| 客户端 | HarmonyOS · ArkTS · ArkUI · Stage 模型 |
| 服务端 | Express · TypeScript · Node.js ≥ 18 |
| 模型 | 默认阿里云百炼兼容接口 · `qwen-plus` |

### 仓库结构

```text
ContractScanner/
├── entry/          # 鸿蒙客户端
├── server/         # 分析服务
├── doc/            # 赛题、选题、交接与样例合同
└── README.md
```

### API（首版）

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/v1/contracts/analyze` | 创建分析任务，返回 `taskId` |
| `GET` | `/v1/tasks/{taskId}/status` | 任务状态 |
| `GET` | `/v1/tasks/{taskId}/result` | 结构化分析结果 |

## 快速开始

### 1. 启动后端

```bash
cd server
cp .env.example .env   # 填入 LLM_API_KEY
npm install
npm run dev            # http://0.0.0.0:3000
```

健康检查：

```bash
curl http://127.0.0.1:3000/health
```

也可用同目录或仓库根目录的 `apikey.txt` 提供密钥（勿提交到 Git）。

### 2. 运行客户端

1. 用 **DevEco Studio** 打开仓库根目录
2. 运行 `entry` 模块到模拟器或真机
3. 默认服务地址：`http://127.0.0.1:3000`
4. **真机联调**：在设置页改为电脑局域网 IP（不要用 `127.0.0.1`）

### 3. 无后端时的演示兜底

导入页可选「高风险样例 / 低风险样例」；设置页支持弱网、超时、服务失败、降级展示等答辩模式。

## 首版边界

当前版本**不做**：电子签署、律师咨询、合同自动改写、多人协作、法律意见书导出。

OCR 为轻量提取 + 兜底；核心交付是大模型结构化解读，而非完整 OCR 产品。
