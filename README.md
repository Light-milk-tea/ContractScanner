# 《法言白话》项目介绍

## 项目简介

《法言白话》是一款面向非法律专业用户的合同智能解读应用，聚焦“看不懂合同”这一高频痛点，帮助用户更快理解合同内容、识别潜在风险，并用更易懂的方式完成重点阅读。

项目基于 HarmonyOS 原生客户端打造，结合服务端大模型分析能力，提供从合同导入、智能解析、风险识别到白话解释的完整体验。

## 项目背景

在租房、求职、兼职、培训、消费、合作等日常场景中，普通用户经常需要面对各类合同或协议。问题并不在于“有没有合同”，而在于：

- 合同内容长，阅读成本高
- 法律表述专业，普通用户难以快速理解
- 风险点分散，容易忽略关键条款
- 用户不知道哪些内容需要重点确认

《法言白话》的目标不是替代律师，而是先帮助普通用户“看明白、看重点、看风险”。

## 解决什么问题

围绕真实使用场景，项目主要解决以下问题：

- 看不懂：把专业合同语言转换成更容易理解的白话表达
- 看不完：先给出整体摘要，帮助用户快速建立认知
- 看不出风险：对条款进行红黄绿分级提示
- 不知道怎么办：对高风险条款给出提醒和修改建议

## 核心能力

### 1. 合同摘要

系统会先对合同内容进行整体分析，提炼出简洁清晰的摘要，帮助用户在短时间内理解合同主题、主要约定和需要重点关注的内容。

### 2. 风险分级

系统会将合同条款按风险程度分为三类：

- 红色：高风险条款，建议重点关注
- 黄色：存在不确定性或需要谨慎确认
- 绿色：相对常规、风险较低

这种分级方式可以帮助用户快速定位关键问题，而不是从头到尾盲目阅读。

### 3. 原文对白话解释

对于每一条重点条款，系统同时展示：

- 合同原文
- 白话解释
- 风险原因
- 修改建议

用户既能看到原始内容，也能快速理解它“实际意味着什么”。

### 4. 智能阅读体验

项目支持从总览到条款详情的分层阅读方式，先看整体，再看重点，最后进入具体条款对照阅读，降低理解门槛。

## 使用流程

用户使用流程简单直接：

1. 导入合同文本或样例内容
2. 等待系统完成分析
3. 查看整体摘要与风险总览
4. 进入重点条款页面查看原文、白话解释和建议

整个主链路已经完整打通，具备稳定演示和使用基础。

## 适用场景

《法言白话》适合以下场景中的普通用户使用：

- 租房合同阅读
- 劳动合同或实习协议阅读
- 培训协议或服务协议确认
- 简单合作协议预读
- 日常消费类条款核对

它尤其适合“没有法律背景，但又希望在签字前先看明白重点”的用户。

## 产品特点

与传统只展示原文的阅读方式相比，《法言白话》更强调“理解效率”和“风险可视化”：

- 用摘要降低整体阅读成本
- 用风险颜色降低识别门槛
- 用白话解释降低专业理解难度
- 用修改建议增强实用价值

项目不是单纯展示合同内容，而是帮助用户建立“先看懂，再决定”的阅读方式。

## 技术形态

项目当前采用“客户端 + 服务端”协作模式：

- 客户端：HarmonyOS 原生应用（ArkTS / ArkUI）
- 服务端：Node.js 分析服务
- 智能分析：通过服务端统一接入 OpenAI 兼容大模型接口

其中，客户端负责交互和展示，模型分析统一由服务端完成，保证整体架构清晰可控。


## 鸿蒙特色能力

作为 HarmonyOS 项目，《法言白话》不仅关注 AI 主链路，也关注全场景体验：

- 应用接续：支持任务状态与阅读过程的连续衔接
- 互动卡片：支持从桌面快捷查看和进入相关内容
- 隐私防窥：在敏感阅读场景下增强信息保护

这些能力让项目不仅是“能用”，也更贴近鸿蒙生态下的实际产品形态。

## 项目边界

为了聚焦首版价值，当前版本暂不包含以下能力：

- 电子签署
- 律师咨询
- 合同自动改写
- 多人协作
- 法律意见书导出

首版重点是把“合同看懂”这件事做好，先完成核心闭环。

## 开发说明

由于项目为鸿蒙应用，开发过程主要结合 Cursor 完成，最终预览与展示主要在 DevEco Studio 模拟器中实现。
