# 前端设计（Frontend Design）

指导产出有辨识度、可上线的前端界面，避免「AI 八股」审美。交付真实可运行代码，并在审美细节与创意选择上做到精致。

用户会给出前端需求：组件、页面、应用或整套界面；可能附带用途、受众或技术约束。

## 设计思考

动手写代码前，先理解语境，并选定一个**大胆、明确**的审美方向：

- **目的**：界面解决什么问题？给谁用？
- **调性**：选一个极端方向，例如：极简暴力、极繁喧闹、复古未来、有机自然、奢华精致、玩具感玩趣、杂志编辑风、粗野原始、装饰艺术/几何、柔和粉彩、工业实用等。可作灵感，但最终只执行**一种**统一方向。
- **约束**：技术要求（框架、性能、无障碍）。
- **差异点**：什么让它难忘？别人会记住的那一件事是什么？

**关键**：选定清晰概念方向后精准执行。大胆极繁与克制极简都成立——关键是**意图明确**，不是「堆得多猛」。

然后实现可运行代码（HTML/CSS/JS、React、Vue 等），要求：

- 生产级、功能完整
- 视觉有冲击、可被记住
- 风格统一、审美立场清楚
- 每个细节都经得起推敲

## 审美指南

聚焦：

- **字体**：选美观、独特、有性格的字体。避免 Arial、Inter 等通用字体；用有辨识度的搭配抬升气质——展示字体要有个性，正文字体要克制。展示字体 + 精炼正文是常用组合。
- **色彩与主题**：承诺一套统一审美。用 CSS 变量保证一致。主色明确 + 锐利点缀，优于平均、怯懦的均分色板。
- **动效**：用动画做氛围与微交互。纯 HTML 优先 CSS；React 可用时用 Motion。抓高影响力时刻：一次编排好的首屏入场（`animation-delay` 错落入场）胜过零散微交互。善用滚动触发与出人意料的 hover。
- **空间构图**：非常规布局。不对称、叠层、斜向流动、打破网格。要么慷慨留白，要么有意控密。
- **背景与视觉细节**：用氛围与纵深，不要默认纯色平涂。加与整体调性匹配的纹理与效果：渐变网格、噪点、几何纹样、层叠透明、戏剧性阴影、装饰边框、自定义光标、颗粒叠加等。

**禁止**千篇一律的 AI 审美：滥用字体族（Inter、Roboto、Arial、系统字体）、陈词滥调配色（尤其白底紫渐变）、可预测布局与组件套路、缺乏语境个性的模具化设计。

创造性解读需求，做真正为语境设计的意外选择。每次设计应不同：明暗主题、字体、调性都要有变化。**禁止**跨项目收敛到同一套常见选择（例如反复用 Space Grotesk）。

**重要**：实现复杂度要匹配审美愿景。极繁需要充分动画与效果；极简/精致需要克制、精准、间距与字体细节。优雅来自把愿景执行到位。

## 落地硬规则

做前端设计任务时，避免通用、堆砌过度的布局。

### 构图与品牌

- **一个构图**：首屏应读成一个整体构图，而不是仪表盘（除非产品本身就是仪表盘）。
- **品牌优先**：品牌页上，品牌/产品名必须是英雄级信号，不能只是导航小字或 eyebrow。任何标题都不应压过品牌。
- **品牌测试**：去掉导航后，若首屏仍可归到别的品牌，说明品牌信号太弱。
- **全出血英雄区**：落地页与推广面，英雄图默认应是占满边缘的主视觉平面或背景。除非既有设计体系明确要求，否则不要用内嵌英雄图、侧栏英雄图、圆角媒体卡片、拼贴拼盘或漂浮图块。
- **首屏预算**：首屏通常只含：品牌、一句标题、一句短支持文、一组 CTA、一张主视觉。不要把数据条、日程、活动列表、地址块、促销、「本周」呼告、元数据行或次要营销塞进首屏。
- **禁止英雄叠加物**：不要在英雄媒体上叠游离标签、漂浮徽章、促销贴纸、信息 chip、呼告框。

### 结构与内容

- **一节一事**：每个区块一个目的、一个标题，通常再加一句短支持文。
- **真实视觉锚点**：图像应展示产品、场所、氛围或语境。装饰性渐变与抽象背景不能充当主视觉。
- **减少杂讯**：避免 pill 簇、数据条、图标行、框式促销、日程碎片、多块抢戏文案。
- **动效用在存在感与层级上**，不要制造噪音。视觉主导作品至少安排 2–3 处有意动效。

### 卡片

- **默认不用卡片**。英雄区禁止卡片。
- 卡片仅在作为**用户交互容器**时允许。若去掉边框/阴影/背景/圆角后不影响交互或理解，就不该做成卡片。

### 色彩与观感

- 选定清晰视觉方向，并定义 CSS 变量。
- **避免** AI 设计常扎堆的套路：
  1. 白底紫 / 紫到靛蓝渐变主题
  2. 暖奶油底（近 `#F4F1EA`）+ 高对比衬线展示字 + 陶土色点缀
  3. 大报版式：细线分割、零圆角、密排报纸栏
- **避免偏好**：默认暗色、紫色、光晕、`rounded-full` 胶囊、多层阴影、emoji。

### 字体与技术

- **字体**：用有表现力、有目的的字体；避免默认栈（Inter、Roboto、Arial、system）。
- **背景**：不要只靠单一纯色底；用渐变、图像或细纹营造氛围。
- **响应式**：桌面与移动端都要能正常加载与阅读。
- **React**：在团队已采用时，可优先现代模式（如 `useEffectEvent`、`startTransition`、`useDeferredValue`）。不要默认加 `useMemo`/`useCallback`，除非项目已有该习惯；遵循仓库的 React Compiler 指引。

### 例外

若在已有网站或设计体系内工作：**保留**既有模式、结构与视觉语言，不要为了「大胆」而推翻体系。

---

不要收敛、不要保守：在框外思考，并完整承诺一个有辨识度的愿景。
